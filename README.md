# OpenWRT WAN IP Checker

Скрипт для OpenWRT, который проверяет IP-адрес WAN-интерфейса и перезапускает его, если адрес не принадлежит заданной сети. Полезно, когда провайдер назначает IP из нежелательного диапазона (например, CGNAT 100.64.0.0/10), а вы хотите получать адреса только из определённой сети (например, 109.108.32.0/19).

## Возможности

- Интерактивная установка с настройкой всех параметров.
- Проверка IP-адреса WAN-интерфейса с заданным интервалом.
- Автоматический перезапуск интерфейса при получении IP не из целевой сети.
- Логирование событий в файл и в системный лог.
- Опциональное логирование попадания в нежелательную сеть (например, CGNAT).
- Автозапуск при загрузке роутера (через `/etc/rc.local`).
- Возможность немедленного запуска после установки.

## Требования

- OpenWRT (проверено на версиях 19.07 и выше).
- Наличие `curl` (установится автоматически, если отсутствует).
- Доступ к интернету для скачивания скрипта.

## Установка одной командой

Выполните на роутере:

```bash
curl -fsSL https://raw.githubusercontent.com/Erridium/openwrt-wan-ip-check/main/install.sh | sh
```
Скрипт установки:

- Скачает основной скрипт `/usr/bin/check_wan_ip.sh`.
- Запросит параметры настройки (можно оставить значения по умолчанию, нажимая Enter).
- Создаст конфигурационный файл `/etc/wan-ip-check.conf`.
- Настроит автозапуск через `/etc/rc.local`.
- Спросит, запустить ли скрипт немедленно.

## Настройка параметров

В процессе установки будут заданы следующие вопросы:

|Параметр|Описание|Пример|
|---|---|---|
|**Имя WAN интерфейса**|Название интерфейса, через который роутер выходит в интернет. Обычно `wan` или `pppoe-wan`.|`wan`|
|**Желаемая сеть (CIDR)**|Сеть, IP из которой считается корректным. Если полученный IP не принадлежит этой сети, интерфейс будет перезапущен.|`109.108.32.0/19`|
|**Нежелательная сеть (CIDR)**|Опционально. Если IP принадлежит этой сети, в лог будет записано предупреждение.|`100.64.0.0/10` (CGNAT)|
|**Интервал проверки (секунд)**|Как часто проверять IP-адрес.|`60`|
|**Задержка после перезапуска интерфейса (секунд)**|Пауза после перезапуска, чтобы интерфейс успел получить новый IP.|`60`|
|**Файл лога**|Путь к файлу для записи логов.|`/var/log/wan_ip_check.log`|

Все параметры сохраняются в `/etc/wan-ip-check.conf`. Вы можете отредактировать этот файл вручную и перезапустить скрипт.

## Управление скриптом

- **Запустить вручную:** `/usr/bin/check_wan_ip.sh &`
- **Остановить:** `killall check_wan_ip.sh`
- **Перезапустить после изменения конфигурации:** сначала остановите, затем запустите заново.

Скрипт автоматически запускается при загрузке роутера (благодаря записи в `/etc/rc.local`). Чтобы отключить автозапуск, удалите или закомментируйте строку `/usr/bin/check_wan_ip.sh &` в файле `/etc/rc.local`.

## Просмотр логов

- **В реальном времени:** `logread -f | grep wan-ip-check`
- **Из файла:** `tail -f /var/log/wan_ip_check.log`
- **Все записи:** `logread | grep wan-ip-check`

Пример вывода:


```text
2025-01-15 10:23:45 - Starting the verification of the WAN interface IP address
2025-01-15 10:23:45 - Target network: 109.108.32.0/19
2025-01-15 10:23:45 - Unwanted network: 100.64.0.0/10
2025-01-15 10:24:45 - Current IP address: 109.108.32.7
2025-01-15 10:24:45 - IP address 109.108.32.7 belongs to the network 109.108.32.0/19
```
## Ручная установка (без install.sh)

Если вы предпочитаете установить вручную:

1. Скачайте основной скрипт:
    
```bash
curl -fsSL -o /usr/bin/check_wan_ip.sh https://raw.githubusercontent.com/Erridium/openwrt-wan-ip-check/main/check_wan_ip.sh
chmod +x /usr/bin/check_wan_ip.sh
```
2. Создайте конфигурационный файл `/etc/wan-ip-check.conf` с нужными параметрами (см. пример выше).
3. Добавьте автозапуск в `/etc/rc.local` перед строкой `exit 0`:
    
    
```bash
/usr/bin/check_wan_ip.sh &
```
2. Запустите скрипт:
```bash
/usr/bin/check_wan_ip.sh &
```

## Возможные проблемы

### 1. Ошибка "curl: (22) The requested URL returned error: 400"

**Причина:** GitHub не может найти файл. Проверьте, что файлы `check_wan_ip.sh` и `install.sh` существуют в репозитории и доступны по прямым ссылкам. Убедитесь, что ветка называется `main` (не `master`).

### 2. Скрипт не запускается при загрузке

Проверьте содержимое `/etc/rc.local`. Должна быть строка `/usr/bin/check_wan_ip.sh &` перед `exit 0`. Убедитесь, что файл `rc.local` исполняемый (`chmod +x /etc/rc.local`).

### 3. Не удаётся получить IP-адрес интерфейса

Убедитесь, что имя интерфейса указано правильно. Проверьте список интерфейсов командой `ifconfig` или `ip addr show`. Если используется PPPoE, интерфейс обычно называется `pppoe-wan`.

### 4. Циклические перезапуски

Если интерфейс перезапускается бесконечно, возможно, целевая сеть задана неверно, или провайдер всегда выдаёт IP из другого диапазона. Увеличьте `RESTART_DELAY` и `CHECK_INTERVAL`, чтобы дать больше времени на получение корректного IP.
